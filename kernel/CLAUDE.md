# monodoc

See `README.md` for orientation, `PRINCIPLES.md` for design commitments,
`RESEARCH.md` for evidence.

## Anti-patterns

- **Don't add a principle without having seen the alternative fail.**
  Principles articulated from theory tend to misfire in ways no one
  notices until much later. Try the wrong thing first; codify only
  after the scar tissue.

- **Don't cite a source without verification.** One attribution in
  RESEARCH.md is already marked unverified ("Eric Lawson / 66 books").
  Mark uncertain findings explicitly rather than asserting them.

- **Don't update the skill at `~/.claude/skills/primer/` from this
  directory.** The skill is downstream of stable principles; regenerate
  it from PRINCIPLES.md when a stable point is reached, not piecemeal
  as patterns evolve.
